Postgres-XC-Installerマニュアル

Postgres-XC-Installerとは?

Postgres-XC-Installer は、Postgres-XCのインストールを簡易にするツールです。
以下の図のように、Postgres-XC-Installerを用いて複数のサーバにPostgres-XCをインストールできます。

プラットフォーム

Postgres-XC-Installerは、以下のソフトウェアをインストールしたCentOS 6.2環境で動作します。

  • JDK 1.7.0
  • Apache Ant 1.7.1

Postgres-XC-Installerは、以下のPostgres-XCバージョンで動作します。

  • Postgres-XC 1.1

注意点

  • Postgres-XC-Installerはrootユーザで実行する必要があります
  • サーバ(インストーラを起動するサーバ、GTMサーバ、Coordinator+Datanodeサーバ)のIPアドレスもしくはホスト名は重複してはいけません
  • Coordinator+Datanodeサーバのノード名は重複してはいけません

起動方法

1, Postgres-XCのソースコード配置

Postgres-XCのソースコードをダウンロードします。ダウンロード先 : Postgres-XC project Page.
TARボールを展開し、postgres-xc-installer/resource/postgres-xcにコピーします。

cp -r postgres-xc postgres-xc-installer/resource/postgres-xc

2, インストーラのビルド

postgres-xc-installer配下でantコマンドを実行します。

cd postgres-xc-installer
ant -f build.xml

3, インストーラの実行

postgres-xc-installer配下でjavaコマンドを実行します。

cd postgres-xc-installer
java -cp bin main/Main

実行例

postgres-xc-installerの実行例を以下に示します。

  1. "Install Postgres-XC"を押します
  1. Postgres-XCをインストールしたいディレクトリを入力します
    注意: ディレクトリは既に存在してはいけません
  2. "Next"を押します
  1. Postgres-XCのスーパーユーザとなるOSユーザを入力します
    注意: このOSユーザは既に存在してはいけません
  2. OSユーザのログインパスワードを入力します
  3. "Next"を押します
  1. GTMが起動するサーバのIPアドレスもしくはホスト名を入力します
  2. GTMのポート番号、ノード名、データディレクトリを入力します
  3. GTMが起動するサーバのrootユーザのログインパスワードを入力します
  4. "Next"を押します
  1. Coordinator+Datanodeが起動するサーバの数を入力します
  2. "Next"を押します
  1. Coordinator+Datanodeが起動するサーバのIPアドレスもしくはホスト名を入力します
  2. Coordinator+Datanodeが起動するサーバのrootユーザのログインパスワードを入力します
  3. GTM-Proxyのポート番号、ノード名、データディレクトリを入力します
  4. Coordinatorのポート番号、ノード名、データディレクトリを入力します
  5. Datanodeのポート番号、ノード名、データディレクトリを入力します
  6. (CoordinatorやDatanodeのpostgresql.confを個別のpostgresql.confにする場合、個別のpostgresql.confを入力します)
  7. Pooler Portを入力します
  8. "Next"を押します
  1. pg_hba.confに記述するサーバ(インストーラを起動するサーバ、GTMサーバ、Coordinator+Datanodeサーバ)のIPアドレスもしくはホスト名を入力します
    詳しくはPostgreSQL Documentationをご覧ください
  2. "Next"を押します
  1. "Next"を押します
  1. "Install"を押します
